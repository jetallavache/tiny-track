# Configuration

See [tinytd Configuration](/tinytd/configuration) for the `[tinytd]`, `[collection]`,
`[storage]`, and `[ringbuffer]` sections.

## `[gateway]`

| Parameter | ENV | Default | Description |
|-----------|-----|---------|-------------|
| `listen` | `TT_LISTEN` | `ws://0.0.0.0:25015` | Listen address and port |
| `update_interval` | `TT_UPDATE_INTERVAL` | `1000` | Push interval, ms |
| `log_backend` | `TT_LOG_BACKEND` | `auto` | `auto` · `stderr` · `stdout` · `docker` · `syslog` · `journal` |
| `log_level` | `TT_LOG_LEVEL` | `info` | Log level |
| `tls_cert` | `TT_TLS_CERT` | — | PEM certificate path |
| `tls_key` | `TT_TLS_KEY` | — | PEM key path |
| `tls_ca` | `TT_TLS_CA` | — | CA bundle (optional) |

## TLS

```ini
[gateway]
listen   = wss://0.0.0.0:25015
tls_cert = /etc/tinytrack/server.crt
tls_key  = /etc/tinytrack/server.key
```

Generate a self-signed certificate:

```bash
openssl req -x509 -newkey rsa:4096 -keyout server.key -out server.crt \
    -days 365 -nodes -subj '/CN=localhost'
```
