# tiny-cli — Overview

**tiny-cli** is the command-line client for TinyTrack. It reads directly from the shared
memory ring buffer — no network connection required. Includes a live ncurses TUI dashboard.

## Commands

```bash
tiny-cli status           # daemon status (tinytd + tinytrack)
tiny-cli metrics          # current snapshot
tiny-cli history l1       # last hour  (1 s resolution)
tiny-cli history l2       # last 24 h  (1 min resolution)
tiny-cli history l3       # last 30 days (1 hr resolution)
tiny-cli dashboard        # live ncurses dashboard
```

## Dashboard panels

| Panel | Description |
|-------|-------------|
| `StatusBar` | Connection state, uptime, hostname |
| `MetricsPanel` | Live metrics with progress bars |
| `TimelinePanel` | ASCII history graph (L1/L2/L3) |
| `LogPanel` | Scrollable event log |
| `HelpBar` | Keyboard shortcuts |

## Keyboard shortcuts (dashboard)

| Key | Action |
|-----|--------|
| `s` | Start / stop streaming |
| `r` | Reset session min/max |
| `1` / `2` / `3` | Switch history level (L1 / L2 / L3) |
| `q` / `Ctrl-C` | Quit |

## Inside Docker

```bash
docker exec -it tinytrack tiny-cli status
docker exec -it tinytrack tiny-cli dashboard
```

## Runtime dependencies

Reads `/dev/shm/tinytd-live.dat` directly via mmap. Requires `libncurses`.
