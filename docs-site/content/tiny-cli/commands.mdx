# Commands

## `tiny-cli status`

Shows the running state of `tinytd` and `tinytrack` systemd services,
PID file contents, and shared memory file presence.

```bash
tiny-cli status
```

## `tiny-cli metrics`

Prints a single current snapshot from the L1 ring buffer.

```bash
tiny-cli metrics
# cpu=23.4%  mem=41.2%  rx=12.1KB/s  tx=3.8KB/s  disk=58.0%  load=0.82
```

## `tiny-cli history`

Dumps history from the specified ring buffer level.

```bash
tiny-cli history l1          # last ~1 hour, 1-second resolution
tiny-cli history l2          # last ~24 hours, 1-minute resolution
tiny-cli history l3          # last ~30 days, 1-hour resolution
tiny-cli history l2 --format=csv   # CSV export (planned)
```

## `tiny-cli dashboard`

Opens the live ncurses TUI. Reads from shared memory at the configured interval.

```bash
tiny-cli dashboard
```

**Keyboard shortcuts:**

| Key | Action |
|-----|--------|
| `s` | Start / stop live update |
| `r` | Reset session min/max |
| `1` | Switch to L1 history |
| `2` | Switch to L2 history |
| `3` | Switch to L3 history |
| `q` / `Ctrl-C` | Quit |

## `tiny-cli logs`

Shows recent log output from `tinytd` and `tinytrack` via journald.

```bash
tiny-cli logs
tiny-cli logs --follow       # tail mode (planned)
```

## `tiny-cli health` *(planned)*

Health check suitable for Docker `HEALTHCHECK`. Returns exit code `0` (ok) or `1` (fail).
Checks shared memory presence and data freshness (timestamp not older than N seconds).

```bash
tiny-cli health
```
