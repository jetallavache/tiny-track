# Architecture

## Component diagram

```
Linux Kernel (/proc/stat, /proc/meminfo, /proc/net/dev, /proc/loadavg, statvfs)
      │
      ▼
  tinytd  ──── epoll + timerfd ────► Ring Buffer Writer (seqlock)
                                           │
                              ┌────────────┴────────────┐
                              ▼                         ▼
                    /dev/shm/tinytd-live.dat    tinytd-shadow.dat
                         (mmap · tmpfs)          (disk · recovery)
                              │
                    ┌─────────┴─────────┐
                    ▼                   ▼
                tinytrack            tiny-cli
              (ws :25015)         (ncurses TUI)
```

## Shared memory layout

The file is a single mmap region structured as:

| Region | Size | Description |
|--------|------|-------------|
| `ttr_header` | 256 bytes | magic, version, writer PID, intervals |
| `ttr_consumer_table` | 2 048 bytes | 32 consumer slots |
| L1 `ttr_meta` | 64 bytes | seq, head, capacity, timestamps |
| L1 data | 3 600 × ~64 bytes ≈ 225 KB | 1-second samples |
| L2 `ttr_meta` | 64 bytes | |
| L2 data | 1 440 × ~64 bytes ≈ 90 KB | 1-minute aggregates |
| L3 `ttr_meta` | 64 bytes | |
| L3 data | 720 × ~64 bytes ≈ 45 KB | 1-hour aggregates |

**Total: ~360 KB** with default settings.

A **seqlock** protects each level from writer/reader races without mutexes.
Readers spin only when a write is in progress (rare, < 1 µs).

## Seqlock protocol

```
Writer:                         Reader:
  seq++  (odd = writing)          seq1 = seq
  write data                      read data
  seq++  (even = done)            seq2 = seq
                                  if seq1 != seq2 → retry
```

## Shadow file

Every `shadow_sync_interval_sec` (default 60 s), tinytd copies the live buffer to
`tinytd-shadow.dat` with an Adler-32 checksum. On startup, if the live buffer is
empty or corrupt, tinytd restores from the shadow file — preserving history across
daemon restarts.
