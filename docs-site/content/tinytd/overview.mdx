# tinytd — Overview

**tinytd** is the metrics collector daemon. It reads Linux kernel interfaces directly,
aggregates data into three-tier ring buffers in shared memory, and makes them available
to `tinytrack` and `tiny-cli` via zero-copy mmap.

## What it collects

| Metric | Source | Description |
|--------|--------|-------------|
| CPU | `/proc/stat` | Total across all cores, % |
| Memory | `/proc/meminfo` | `(total − available) / total`, % |
| Network RX/TX | `/proc/net/dev` | All interfaces except `lo`, bytes/s |
| Disk | `statvfs(rootfs_path)` | Root filesystem usage, % |
| Load average | `/proc/loadavg` | 1 / 5 / 15 minutes |

## Ring buffer tiers

| Level | Resolution | Capacity | Covers |
|-------|-----------|----------|--------|
| L1 | 1 second | 3 600 entries | ~1 hour |
| L2 | 1 minute | 1 440 entries | ~24 hours |
| L3 | 1 hour | 720 entries | ~30 days |

L1 is aggregated into L2 every 60 s; L2 into L3 every 3 600 s.
The buffer lives in `/dev/shm` (tmpfs) — zero-copy mmap access.
It is periodically synced to a shadow file on disk for recovery after restart.

## Resource usage

- CPU: < 1% on a single-core VPS
- RAM: < 10 MB
- Shared memory: ~360 KB with default capacities

## Runtime dependencies

Only `libc`. No external libraries required.
