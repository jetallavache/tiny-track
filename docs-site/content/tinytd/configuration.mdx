# Configuration

## Priority order

`hardcoded defaults` → `tinytrack.conf` → `ENV variables` → `CLI arguments`

Each layer overrides the previous. In Docker, the entrypoint generates a config from
defaults and then patches it with ENV values.

## Config files

| File | Purpose |
|------|---------|
| `etc/tinytrack.conf` | Production (host) |
| `etc/tinytrack.conf-docker` | Docker / container |
| `etc/tinytrack.conf-debug` | Local debug |
| `tests/tinytrack.conf-test` | Automated tests |

## `[tinytd]`

| Parameter | Default | Description |
|-----------|---------|-------------|
| `user` | `tinytd` | User after privilege drop |
| `group` | `tinytd` | Group |
| `pid_file` | `/var/run/tinytd.pid` | Path to PID file |
| `log_backend` | `auto` | `auto` · `stderr` · `stdout` · `docker` · `syslog` · `journal` |
| `log_level` | `info` | Minimum log level |

> Use `docker` backend in containers — it writes to stdout without a timestamp.
> Docker adds its own timestamp via `--log-opt`.

## `[collection]`

| Parameter | ENV | Default | Description |
|-----------|-----|---------|-------------|
| `interval_ms` | `TT_INTERVAL_MS` | `1000` | Collection interval, ms |
| `du_interval_sec` | `TT_DU_INTERVAL_SEC` | `30` | Disk usage refresh, s |
| `proc_root` | `TT_PROC_ROOT` | `/proc` | Path to `/proc` |
| `rootfs_path` | `TT_ROOTFS_PATH` | `/` | Path to root filesystem |

## `[storage]`

| Parameter | ENV | Default | Description |
|-----------|-----|---------|-------------|
| `live_path` | `TT_LIVE_PATH` | `/dev/shm/tinytd-live.dat` | Live ring buffer (tmpfs) |
| `shadow_path` | `TT_SHADOW_PATH` | `/var/lib/tinytrack/tinytd-shadow.dat` | Persistent shadow copy |
| `shadow_sync_interval_sec` | — | `60` | Sync interval, s |
| `file_mode` | — | `416` (0640) | File permissions (decimal) |

> `live_path` must be on tmpfs (`/dev/shm`). In Docker, use a distinct filename
> (e.g. `tinytd-docker-live.dat`) to avoid conflicts with a host daemon sharing `/dev/shm`.

## `[ringbuffer]`

| Parameter | ENV | Default | Description |
|-----------|-----|---------|-------------|
| `l1_capacity` | `TT_L1_CAPACITY` | `3600` | L1 entries |
| `l2_capacity` | `TT_L2_CAPACITY` | `1440` | L2 entries |
| `l3_capacity` | `TT_L3_CAPACITY` | `720` | L3 entries |
| `l2_agg_interval_sec` | `TT_L2_AGG_INTERVAL` | `60` | L1→L2 aggregation interval |
| `l3_agg_interval_sec` | `TT_L3_AGG_INTERVAL` | `3600` | L2→L3 aggregation interval |
