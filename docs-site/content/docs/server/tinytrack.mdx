# tinytrack вЂ” Gateway

`tinytrack` is the WebSocket and HTTP gateway. It reads metrics from shared memory
and serves them to clients using binary protocol v2.

## Architecture

On startup, `tinytrack`:

1. Opens the shared memory segment written by `tinytd`
2. Binds a TCP socket (default port 25015)
3. Optionally loads a TLS certificate for WSS/HTTPS
4. Enters an epoll event loop, accepting connections and serving requests

The gateway is single-threaded and uses non-blocking I/O with epoll for high concurrency.

## Endpoints

| Endpoint | Protocol | Description |
|----------|----------|-------------|
| `/v1/stream` | WebSocket | Binary protocol v2 metrics stream |
| `/websocket` | WebSocket | Alias for `/v1/stream` (legacy) |
| `/metrics` | HTTP | Prometheus text format |
| `/health` | HTTP | Health check (`200 OK` if running) |

## Command-line flags

```
tinytrack [OPTIONS]

  -c FILE    Configuration file (default: /etc/tinytrack/tinytrack.conf)
  -f         Run in foreground
  -v         Verbose logging
  -d         Debug mode
  -V         Print version and exit
  -h         Print help and exit
```

## Configuration

```ini
[gateway]
port          = 25015
bind_addr     = 0.0.0.0
max_clients   = 64
tls_cert      = /etc/tinytrack/certs/server.crt
tls_key       = /etc/tinytrack/certs/server.key
tls_enabled   = false
```

See [Configuration](/docs/server/configuration) for the full reference.

## TLS / WSS

To enable TLS:

```ini
[gateway]
tls_enabled = true
tls_cert    = /etc/tinytrack/certs/server.crt
tls_key     = /etc/tinytrack/certs/server.key
```

Self-signed certificates for development are included in `server/etc/certs/`.

## Debugging

```bash
tinytrack -f -v -c /etc/tinytrack/tinytrack.conf
```

## Troubleshooting

See [Troubleshooting](/docs/server/troubleshooting) for common issues.

Common problems:

- **Port already in use**: change `port` in configuration or stop the conflicting process
- **No metrics**: verify `tinytd` is running and the shared memory path matches
- **TLS errors**: check certificate paths and permissions
