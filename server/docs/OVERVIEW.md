# TinyTrack — Обзор

## Что это

TinyTrack — демон сбора системных метрик для Linux с real-time стримингом через WebSocket. Состоит из трёх компонентов:

- **tinytd** — собирает метрики из `/proc` и `/sys`, пишет в кольцевой буфер в shared memory
- **tinytrack** — читает буфер и стримит метрики клиентам по WebSocket/HTTP
- **tiny-cli** — CLI-клиент для просмотра метрик и управления

## Зачем

- Мониторинг хоста или контейнера без агентов на стороне клиента
- Real-time дашборд в браузере или терминале
- История метрик трёх уровней детализации (1ч / 24ч / 7д)
- Минимальные зависимости: только libc + libssl в рантайме
- Работает в Docker с мониторингом **хостовой** системы через bind-mount `/proc`

## Что собирается

| Метрика | Источник | Описание |
|---------|----------|----------|
| CPU usage | `/proc/stat` | Суммарная загрузка всех ядер, % |
| Memory usage | `/proc/meminfo` | (total − available) / total, % |
| Network RX/TX | `/proc/net/dev` | Суммарный трафик всех интерфейсов кроме lo, байт/с |
| Disk usage | `statvfs(rootfs_path)` | Использование корневой ФС, % |
| Load average | `/proc/loadavg` | 1/5/15 минут |

## Как использовать

### Быстрый старт на хосте

```bash
./bootstrap.sh && ./configure && make
sudo make install
sudo systemctl start tinytd tinytrack
```

Подключитесь к `ws://localhost:25015/websocket` любым WebSocket-клиентом.

### Быстрый старт в Docker

```bash
docker compose up -d
docker compose exec tinytrack tiny-cli dashboard
```

### Подключение клиента

Gateway принимает WebSocket-соединения на порту `25015` (по умолчанию).

- **WebSocket**: `ws://host:25015/websocket`
- **HTTP snapshot**: `GET http://host:25015/api/metrics/live`
- **TLS**: `wss://host:25015/websocket` (если настроен TLS)

После подключения сервер автоматически:
1. Отправляет `PKT_CONFIG` с текущими настройками
2. Начинает пушить `PKT_METRICS` каждые `interval_ms` мс

## Кольцевой буфер

Метрики хранятся в трёх кольцах с разной детализацией:

| Уровень | Разрешение | Ёмкость по умолчанию | Покрытие |
|---------|-----------|----------------------|----------|
| L1 | 1 сек | 3600 записей | ~1 час |
| L2 | 1 мин (агрегация L1) | 1440 записей | ~24 часа |
| L3 | 1 час (агрегация L2) | 720 записей | ~30 дней |

Буфер хранится в `/dev/shm` (tmpfs) — zero-copy доступ через mmap. Периодически синхронизируется в shadow-файл на диске для восстановления после перезапуска.
