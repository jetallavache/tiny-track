Спецификация бинарного протокола мониторинга

## 🔹 Заголовок пакета

|Поле|Размер|Тип|Описание|
|---|---|---|---|
|`version`|1 байт|`uint8_t`|Версия протокола (сейчас `1`).|
|`type`|1 байт|`uint8_t`|Тип пакета (см. ниже).|
|`length`|2 байта|`uint16_t`|Длина payload (big endian, без заголовка).|

---

## 🔹 Типы пакетов (`type`)

|Код|Имя|Направление|Назначение|
|---|---|---|---|
|`1`|`PKT_METRICS`|Сервер → клиент|Передача метрик системы.|
|`2`|`PKT_CONFIG`|Сервер → клиент|Отправка текущей конфигурации.|
|`3`|`PKT_ALERT`|Сервер → клиент|Алерты (например, CPU > 90%).|
|`4`|`PKT_CMD`|Клиент → сервер|Команды управления (изменение интервала, включение алертов).|
|`5`|`PKT_ACK`|Сервер → клиент|Подтверждение выполнения команды.|

---

## 🔹 Payload по типам пакетов

### 1. `PKT_METRICS` (сервер → клиент)

|Поле|Размер|Тип|Описание|
|---|---|---|---|
|`cpu_usage`|4 байта|`float`|Загрузка CPU (%)|
|`mem_usage`|4 байта|`float`|Использование памяти (%)|
|`loadavg1`|4 байта|`float`|Load average (1 минута)|
|`loadavg5`|4 байта|`float`|Load average (5 минут)|
|`loadavg15`|4 байта|`float`|Load average (15 минут)|
|`net_rx`|8 байт|`uint64_t`|Принято байт|
|`net_tx`|8 байт|`uint64_t`|Отправлено байт|

**Общий размер:** 36 байт.

---

### 2. `PKT_CONFIG` (сервер → клиент)

|Поле|Размер|Тип|Описание|
|---|---|---|---|
|`interval_ms`|4 байта|`uint32_t`|Интервал обновления метрик (мс).|
|`alerts_enabled`|1 байт|`uint8_t`|0 = выкл, 1 = вкл.|

**Общий размер:** 5 байт.

---

### 3. `PKT_ALERT` (сервер → клиент)

|Поле|Размер|Тип|Описание|
|---|---|---|---|
|`level`|1 байт|`uint8_t`|Уровень (0=info, 1=warn, 2=crit).|
|`message`|64 байта|`char[64]`|Сообщение (null-terminated, фикс. длина).|

**Общий размер:** 65 байт.

---

### 4. `PKT_CMD` (клиент → сервер)

|Поле|Размер|Тип|Описание|
|---|---|---|---|
|`cmd_type`|1 байт|`uint8_t`|Тип команды (см. ниже).|
|`data`|N байт|-|Данные в зависимости от команды.|

#### Подтипы команд (`cmd_type`):

|Код|Название|Payload|
|---|---|---|
|`1`|`SET_INTERVAL`|`uint32_t interval_ms`|
|`2`|`SET_ALERTS`|`uint8_t enabled` (0/1)|

---

### 5. `PKT_ACK` (сервер → клиент)

|Поле|Размер|Тип|Описание|
|---|---|---|---|
|`cmd_type`|1 байт|`uint8_t`|Какой тип команды подтверждён.|
|`status`|1 байт|`uint8_t`|0=OK, 1=ERROR.|

**Общий размер:** 2 байта.

---

## 🔹 Пример обмена

1. **Подключение**
    
    - Клиент открывает WS.
        
    - Сервер → `PKT_CONFIG { interval=1000, alerts=1 }`.
        
2. **Смена интервала**
    
    - Клиент → `PKT_CMD { cmd_type=1, interval=2000 }`.
        
    - Сервер → `PKT_ACK { cmd_type=1, status=0 }`.
        
3. **Стрим метрик**
    
    - Сервер → `PKT_METRICS { cpu=23.5, mem=41.2, ... }` каждые 2 сек.
        
4. **Алерт**
    
    - Сервер → `PKT_ALERT { level=2, "CPU > 90%" }`.
        


## 🔹 Рекомендации

- Все **многобайтовые числа** передавать в **big endian** (сеть → одинаково на любых архитектурах).
    
- Строки фиксированной длины (`message`) — всегда null-terminated, остальное заполнять нулями.
    
- Клиент и сервер должны проверять `version` и `length`, чтобы не упасть при несовместимости.
